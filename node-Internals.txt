1- What is the Node.js Event Loop?

ال event loop هو المحرك الاساسي او القلب بتاع node.js اللي فيه بتتم العمليات ال async بدون ما يحصل block لل main therad ودوره انه بيفضل يتابع ال call stack و ال callback queue واول لما ال call stack يفضي يشوف لو في callbacks جاهزه يبدأ ينفذها

2- What is Libuv and What Role Does It Play in Node.js?

ال libuv هو c library موجوده في ال node.js بتتعامل مع ال I/O operations وبتدير ال therad pool اللي فيه ال virtual therads بتاخد العمليات ال asyc من ال call stack عشان تنفذها عشان تخليه فاضي بحيث ميحصلش block لل main therad

// ******************************************************//

3- How Does Node.js Handle Asynchronous Operations Under the Hood?

اول حاجه ال main therad بياخد العمليه ال async ويروح يبعتها لل libuv بعدين ال libuv بتاخد العمليه دي ي اما تبعتها لل os kernel يعملها بشكل تلقائي زي ال network requests ي اما تبعتها لل virtual therads تعملها ولما ال callback بتاعها يجهز يقوم ال event loop واخدها ومرجعها لل call stack عشان تتنفذ 

// ******************************************************//

4- What is the Difference Between the Call Stack, Event Queue, and Event Loop in Node.js?

ال call stack هو المكان اللي بيتنفذ فيه كود ال js العادي خطوه بخطوه 
ال event queue  هو الطابور اللي بتقف فيه ال call backs بتاعت العمليات ال async اللي خلصت ومستنيه عشان تتنفذ 
ال event loop دا اللوب اللي بيفضل يلف علي ال event queue ويتابع ال call stack واول لما يفضي يبعتله اول call back جاهزه عشان تتنفذ

5- What is the Node.js Thread Pool and How to Set the Thread Pool Size?

ال thread pool عبارة عن مجموعة من ال virtual therads موجوده في ال libuv بيبقي عددهم الافتراضي 4 ودول بيعملو العمليات ال async اللي بتاخد وقت 
واقدر اغير عددهم عن طريق اني بروح اغير ال variable اللي اسمه UV_THREADPOOL_SIZE من خلال command line بكتبه في ال terminal وانا بشغل الملف بتاعي زي مثلا ==> UV_THREADPOOL_SIZE=8 node index.js

6- How Does Node.js Handle Blocking and Non-Blocking Code Execution?

في العمليات ال sync (blocking code) الكود بيتنفذ خطوه خطوه علي ال main single therad ولو في عمليه بتاخد وقت بيفضل مستنيها ويوقف الكود لحد ما تخلص ودا اللي بيعمل block لل main therad ويخليه ميقدرش يستقبل اي طلبات تاني لحد ما العمليه دي تتنفذ 

انما في العمليات ال async (non-blocking code) ال main therad بياخد العمليه ال async ويبعتها لل libuv تديها لل virtual therads يخلصوها في الخلفيه ويكمل هو الكود بدون انتظار ولما ال call back بتاعها يجهز يرجعله عشان ينفذه ودا اللي بيخليه سريع وبيعمل عمليات كتيره في نفس الوقت
